## 方法1：Android Studio 模拟器（强烈推荐）⭐⭐⭐⭐⭐

### 为什么用模拟器？
- ⚡ **最快**：5-10秒部署，无需USB连接
- 🎯 **即时预览**：改代码 → 点击Run → 立即看效果
- 📷 **完整功能**：相机、录音、GPS等全部支持
- 🔄 **热更新**：支持Apply Changes，无需重启App

### 设置模拟器

#### 1. 打开 Android Studio
```bash
# 如果你已经有Android Studio，直接打开项目
cd android_translator_app
# 用Android Studio打开当前目录
```

#### 2. 创建模拟器（AVD）

**步骤**：
1. 点击顶部工具栏的 `AVD Manager` 图标（手机图标）
2. 点击 `Create Virtual Device`
3. 选择设备类型：
   - 推荐：**Pixel 5** 或 **Pixel 6**
   - 屏幕尺寸：5.5-6.5英寸
4. 选择系统镜像：
   - 推荐：**Android 11 (API 30)** 或 **Android 12 (API 31)**
   - 点击 `Download` 下载镜像（首次需要）
5. 点击 `Finish`

**配置建议**：
```
设备名称: Pixel 5 API 30
Android版本: Android 11 (API 30)
CPU: x86_64
RAM: 2048 MB
内部存储: 2048 MB
```

#### 3. 启动模拟器

**在Android Studio中**：
1. 点击顶部工具栏的设备下拉框
2. 选择刚创建的模拟器
3. 点击绿色的 ▶️ **Run** 按钮

**等待10-20秒，模拟器启动！**

#### 4. 运行App

**自动部署**：
- 点击Run后，Android Studio会：
  1. 编译项目
  2. 安装到模拟器
  3. 自动启动App

**整个过程只需 5-10秒！**

## 方法2：Apply Changes（增量更新）⭐⭐⭐⭐

### 更快的调试方式！

**功能**：只更新改动的代码，无需重新安装

**使用**：
1. 修改代码（如改个文字、调整UI）
2. 点击 `Apply Changes and Restart Activity` 图标（⚡闪电图标）
3. **2-3秒**即可看到效果！

**快捷键**：
- `Ctrl + F10` (Windows/Linux)
- `Cmd + F10` (Mac)

**适用场景**：
- ✅ 修改UI文字、颜色、尺寸
- ✅ 调整布局
- ✅ 修改简单逻辑
- ❌ 添加新Activity（需要完整Run）
- ❌ 修改AndroidManifest

## 方法3：无线调试（Wireless ADB）⭐⭐⭐

### 不想插USB线？用WiFi连接！

**前提条件**：
- 手机和电脑在同一WiFi网络
- Android 11+ 手机

**步骤**：

#### 1. 手机端设置
```
设置 → 关于手机 → 连续点击"版本号" 7次 → 开发者选项已启用
设置 → 系统 → 开发者选项 → 启用"无线调试"
```

#### 2. 配对设备（首次）
```bash
# 在手机上点击"使用配对码配对设备"，会显示IP和端口
# 例如：192.168.1.100:12345 配对码：123456

# 在电脑上运行
adb pair 192.168.1.100:12345
# 输入配对码：123456

# 连接设备
adb connect 192.168.1.100:54321  # 使用手机显示的实际IP:端口
```

#### 3. 验证连接
```bash
adb devices
# 应该看到：
# 192.168.1.100:54321    device
```

#### 4. 在Android Studio中运行
- 现在设备下拉框会显示你的手机
- 点击Run，通过WiFi部署！

**好处**：
- ✅ 不用插USB线
- ✅ 可以边充电边调试
- ✅ 速度和USB相当

## 方法4：Scrcpy（投屏 + 控制）⭐⭐⭐⭐

### 把手机屏幕投到电脑上！

**特点**：
- 📺 手机屏幕实时投到电脑
- 🖱️ 用鼠标键盘控制手机
- ⚡ 超低延迟
- 📋 复制粘贴互通

**安装**：
```bash
# Ubuntu/Debian
sudo apt install scrcpy

# macOS
brew install scrcpy

# Windows
# 下载：https://github.com/Genymobile/scrcpy/releases
```

**使用**：
```bash
# USB连接手机后
scrcpy

# 或者无线（需先配对）
scrcpy --tcpip=192.168.1.100:5555
```

**配合开发**：
1. 手机USB连接（或无线ADB）
2. 运行scrcpy投屏到电脑
3. Android Studio部署到手机
4. 在电脑大屏幕上操作测试
5. 可以录屏、截图

## 模拟器技巧

### 1. 模拟相机
模拟器会使用虚拟相机：
- 前置：动画图案
- 后置：3D场景（可以用鼠标旋转）

**测试拍照翻译**：
- 可以导入图片到模拟器
- 拖拽图片到模拟器窗口
- 然后在相册中选择

### 2. 模拟录音
模拟器支持麦克风输入：
- Settings → Microphone
- 选择电脑麦克风
- 可以真实录音测试

### 3. 快速重置
如果模拟器出问题：
```
AVD Manager → 右键模拟器 → Wipe Data
```

### 4. 快照功能
保存模拟器状态，下次秒开：
```
模拟器启动时会自动保存快照
下次启动直接恢复到上次状态（2-3秒启动）
```

## 常见问题

### Q: 模拟器很卡？
**A: 优化方法**
```
1. 启用硬件加速：
   - Windows: 安装 Intel HAXM
   - Mac: 自动支持
   - Linux: 启用 KVM

2. 减少RAM分配：
   - AVD Manager → Edit → Advanced Settings
   - RAM: 2048 MB（不要设太高）

3. 选择x86镜像（不要用ARM）
```

### Q: 模拟器没有Play Store？
**A: 选择带Play Store的镜像**
```
创建AVD时选择：
- Play Store图标的系统镜像（Google APIs）
- 不选择 Google Play 的话没有Play商店
```

### Q: Apply Changes不生效？
**A: 某些改动需要完整Run**
```
需要完整Run的情况：
- 添加新Activity/Fragment
- 修改AndroidManifest.xml
- 添加新的依赖
- 修改native代码

此时点击普通的 Run 按钮即可
```

### Q: 无线调试连接不上？
**A: 检查步骤**
```bash
# 1. 确保在同一WiFi
ping 192.168.1.100  # 手机IP

# 2. 重新配对
adb kill-server
adb start-server
adb pair xxx:xxxxx

# 3. 重启手机的无线调试
```
